# Ash Nazg frontend

Vue 3 + TypeScript bundles that integrate Ash Nazg into Nextcloud's
Files app and admin settings page.

## Layout

```
frontend/
├── package.json              ← deps; engines.node = >=22
├── vite.config.ts            ← multi-entry, outputs to ../host/static
├── tsconfig.json             ← strict: true; vue-tsc-checked
├── tsconfig.node.json        ← config-file scope
├── .eslintrc.cjs             ← @nextcloud preset + project guards
├── env.d.ts                  ← Vite + Vue SFC type shims
└── src/
    ├── files-action.ts       ← bundle: files-action — registers the right-click action
    ├── admin-settings-main.ts← bundle: admin-settings — mounts AdminSettings.vue
    ├── AdminSettings.vue     ← the admin panel
    └── IframeHost.vue        ← placeholder for the KasmVNC stream container
```

Build outputs into `../host/static/` (where the host container serves
them from). Filenames carry a content hash; the host reads the
generated `manifest.json` to inject the right `<script src="...">` per
template.

## Installing dependencies

**This repo deliberately does not run `npm install`.** Per the
project's supply-chain rules (see `CONTRIBUTING.md` and the
repo-root `CLAUDE.md`):

```bash
cd frontend
npm ci --ignore-scripts          # reproducible from package-lock.json
npm audit signatures              # verify registry signatures
git diff package-lock.json        # review the lockfile diff before commit
```

The first run will need a `package-lock.json` to be created. The
boring-and-auditable bootstrap path:

```bash
# one-time, on a clean checkout, by a human (not by an agent)
npm install --ignore-scripts --package-lock-only
git add package-lock.json
git diff --staged package-lock.json   # eyeball it
```

After that, every CI build and every developer install uses
`npm ci --ignore-scripts`. Postinstall scripts stay disabled by
default. Re-enable individual packages only after explicit review.

## Type checking and lint

```bash
npm run typecheck          # vue-tsc --noEmit
npm run lint               # ESLint over src/
npm run build              # typecheck + vite build
```

`tsconfig.json` runs in **strict** mode with `noUnusedLocals`,
`noUnusedParameters`, `noImplicitReturns`, and
`exactOptionalPropertyTypes`. Don't relax these without an OpenSpec
change documenting why.

## What is wired in this scaffold

- The Files action surface (extension + size predicate, admin gate,
  toast + event-bus emit on click).
- The admin settings panel (toggle + numeric inputs + Save and
  Test buttons; both stubs that toast).
- The iframe-host component as a placeholder.

## What is intentionally NOT here

- No actual `/run` HTTP call from the action (lands in
  `wire-dosbox-engine`).
- No real KasmVNC client (lands in `streaming-proxy`).
- No persistence for the admin settings (lands in
  `admin-settings-ui` follow-up change).
- No SVG icon for the file action.
- No translation files (`l10n/*.json`) — these are generated by
  the build pipeline in a later change; the strings here are
  already wrapped in `t('ash_nazg', ...)` so they will be picked
  up when extraction runs.

## Versions and compatibility

- Vue 3.4+ (Nextcloud 30 era)
- Vite 5.x (matches Nextcloud 30 frontend tooling)
- TypeScript 5.4+
- vue-tsc 2.x
- Node 22 LTS

`@nextcloud/*` packages are pinned to caret-major ranges that match
Nextcloud 30. The lockfile (created on first install) is the source
of truth for exact versions.
